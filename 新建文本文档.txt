items_game.txt具有以下结构：
"items_game"
{
    ......
    "items"
    {
        ......
        "foo"
        {
            ......
        }
        ......
    }
    ......
}



我需要一个通过配置脚本处理文本的程序，该程序满足：
存在一个toml格式的配置文件，配置文件中每个单元都会指定一个目录（如items_game/items/bar）和一段文字。
每个单元即将bar的子括号里的内容替换为该文字（保持bar在items目录中的次序而非添加到最后）。程序会依次执行每个单元，尽量用已经成熟的方案，在形式上可以更规范而非一定要按照我的格式。
